    SetupPosware

    Prima revisione documento: 27 agosto 2024
    Ultima revisione documento: 28 agosto 2024

    Cenni preliminari

    Posware non aveva più un setup da ormai molto tempo, motivo per il quale se si vuole installare Posware Frontend da zero, su una macchina Windows che non ha precedenti versioni di Posware 4.x installate, non era di fatto possibile procedere. L'unico setup esistente, infatti, risale alla versione 2.x ed è ormai obsoleto e deprecato. Per questo motivo, a partire dalla versione 4.3, si ha un nuovo setup riprogettato per poter installare Posware Frontend da zero.

    Il presente documento, come introdotto nell'overview generale, descrive dettagliatamente il funzionamento di questo nuovo setup di Posware Frontend, il SetupPosware, analizzando in particolare le differenze presenti nel suo utilizzo e nei passaggi preliminari da eseguire tra lo scenario di installazione ex novo e lo scenario di migrazione dalla versione 4.2 alla 4.3 di Posware. Inoltre, vengono illustrate una serie di informazioni utili riguardanti l'aggiornamento dei file di configurazione di Posware a seguito dell'esecuzione con successo del setup.

    Prerequisiti

    Prima di procedere con l'utilizzo del SetupPosware assicurarsi di avere le risorse hardware minime indispensabili indicate nell'overview generale e di aver installato almeno le dipendenze necessarie con installazione manuale.

    Scenari d'uso

    Il pacchetto di installazione SetupPosware copre unicamente questi due scenari:

    • Installazione ex novo
    • Migrazione dalla versione 4.2 alla 4.3

    Per quanto riguarda invece gli aggiornamenti della Build version, quindi da una versione precedente della 4.3 all'ultima 4.3 aggiornata, questi rimangono sempre in carico al PosUpdate.

    In entrambi gli scenari l'utente può lanciare il setup facendo ordinario doppio click sull'applicativo. In questo caso nessun parametro particolare viene passato al SetupPosware e verrà mostrata l'interfaccia principale da cui l'utente potrà procedere con l'installazione seguendo il wizard.

    Oltre al classico doppio click, è possibile avviare l'applicativo anche tramite terminale passandogli alcuni dei parametri standard previsti da Inno Setup:

    • /SILENT, /VERYSILENT
    • /LOG
    • /LOG="filename"
    • /NOCANCEL

    Per la lista completa dei parametri standard previsti da Inno Setup, così come per sapere cosa fanno quelli soprastanti, consultare la guida ufficiale di Inno Setup.

    Oltre a quelli standard, inoltre, è stato introdotto anche un parametro custom esclusivo per il SetupPosware:

    • /SKIPOPOS

    Questo parametro indica di non scaricare ed installare l'ultima versione dei Monroes OPOS Common Control Objects (v. 1.14.001). Questo parametro è utile in caso di incompatibilità dei driver OPOS in uso su RT-ONE o NCR. Per quanto riguarda, invece, EPSON è da noi certificato il funzionamento.

    Installazione ex novo

    Prima di poter lanciare il SetupPosware e procedere con l'installazione ex novo di Posware Frontend è necessario assicurarsi ancora nuovamente di aver installato almeno le dipendenze di terze parti che necessitano di installazione manuale. In particolare:

    1. Verificare il sistema operativo:
      • Se si utilizza Windows 7 SP1 appena installato e mai aggiornato, consulta il seguente paragrafo.
      • Se si utilizza una versione obsoleta di Windows 10, il setup darà un errore specifico. La versione minima utilizzabile è riportata nei requisiti minimi dell'overview generale. Esegui l'aggiornamento di Windows 10 e riprova.
      • Se si utilizza Windows 11 non ci sono particolari accorgimenti di cui tenere conto a livello di sistema operativo.
    2. Assicurarsi di aver installato .NET Framework v. 4.8, altrimenti il SetupPosware darà errore.
    3. Accertarsi di aver installato MySQL v. 5.6 e che sia accessibile con le nuove credenziali complesse, altrimenti il setup darà un errore specifico e non farà continuare l'installazione.

    NOTA BENE: Si ricorda che nelle nuove installazioni di Posware Frontend NON è più possibile utilizzare MySQL v. 4.1

    Se tutte le condizioni di cui sopra sono soddisfatte, il SetupPosware si avvierà e procederà con l'installazione del nuovo Posware Frontend.

    In questo scenario, verrà installato un nuovo database standard pronto all'uso che potrà poi essere personalizzato e in cui andrà inviata tutta l'anagrafica del punto vendita.

    NOTA BENE: Una volta conclusa l'installazione, il database NON potrà assolutamente essere sostituito con uno esistente di una versione 4.2.x di Posware, perché incompatibili. Se è necessario effettuare un'installazione pulita usando però un database precedente come base, consultare il paragrafo successivo.

    Usare un database di una versione 4.2.x di Posware in una installazione ex novo

    È possibile utilizzare un database cassa esistente in una installazione ex novo a patto che provenga da una versione di Posware 4.2.306 o superiore. Il database dovrà essere dumpato e ripristinato su MySQL v. 5.6 per usarlo come base di installazione per Posware 4.3. È possibile usare Navicat per questa operazione, ma vista la possibilità di errori di migrazione, è preferibile usare il tool ufficiale mysqldump.

    L'operazione di dump e ripristino deve essere effettuata assolutamente PRIMA di lanciare il setup. Il nome del database deve essere sempre cassa e le credenziali MySQL v. 5.6 devono essere le nuove credenziali complesse.

    Il setup verificherà se il database è compatibile. In caso positivo procederà con il suo aggiornamento e poi con l'installazione. In caso negativo verrà notificato un errore specifico.

    Migrazione dalla versione 4.2 alla 4.3

    Prima di poter lanciare il SetupPosware e procedere con la migrazione dalla versione 4.2 alla 4.3 di Posware Frontend è necessario completare tutti i seguenti controlli e le operazioni di backup, da usare in caso di malfunzionamenti e per l'eventuale procedura di rollback. In particolare:

    1. Effettuare il backup sia del database cassa che di tutta la cartella di Posware.

      NOTA BENE: IL PROCESSO DI INSTALLAZIONE/AGGIORNAMENTO NON È REVERSIBILE ED IL DATABASE cassa UNA VOLTA AGGIORNATO ALLO SCHEMA 4.3 NON È PIÙ COMPATIBILE CON POSWARE v. 4.2

    2. Assicurarsi di avere una versione di Posware installata e funzionante non inferiore alla v. 4.2.306.

    3. Verificare il sistema operativo:
      • Se si utilizza Windows 7 SP1 appena installato e mai aggiornato, consulta il seguente paragrafo.
      • Se si utilizza una versione obsoleta di Windows 10, il setup darà un errore specifico. La versione minima utilizzabile è riportata nei requisiti minimi dell'overview generale. Esegui l'aggiornamento di Windows 10 e riprova.
      • Se si utilizza Windows 11 non ci sono particolari accorgimenti di cui tenere conto a livello di sistema operativo.
    4. Accertarsi che la versione installata di MySQL (MySQL v. 4.1 o v. 5.6) sia avviata ed accessibile e con il database cassa presente al suo interno, altrimenti il setup darà un errore specifico e non farà continuare l'installazione.

    Se tutte le condizioni di cui sopra sono soddisfatte, il SetupPosware si avvierà e procederà con l'installazione del nuovo Posware Frontend. Innanzitutto, all'avvio il SetupPosware verificherà i prerequisiti di migrazione dalla versione precedente di Posware. Qualora l'installazione precedente fosse corrotta o con dati mancanti, si riceverà un errore e non sarà possibile continuare.

    Al contrario dello scenario di installazione ex novo, il SetupPosware non bloccherà l'installazione se rileva una versione di MySQL v. 4.1 nella macchina. Questo per evitare di far migrare senza vantaggi concreti la maggior parte delle installazioni di Posware v. 4.2 che usano MySQL v. 4.1 come database.

    Una volta terminata l'installazione, come anticipato, il database sarà migrato alla versione 4.3 con gli schemi aggiornati e non sarà più compatibile con la versione 4.2 di Posware. Quindi è bene effettuare un backup prima di eseguire qualsiasi operazione di migrazione.

    La migrazione alla v. 4.3 effettua anche la migrazione e l'aggiornamento di file esistenti, quali i file di configurazione di Posware, delle Connection strings e delle impostazioni utente. Per ulteriori dettagli consultare il relativo paragrafo.

    Infine, dopo aver completato con successo l'intero processo, potrebbe essere necessario riavviare il computer per iniziare ad usare Posware v. 4.3. In questo caso, NON usare assolutamente il software prima di aver riavviato il sistema.

    Eventuali errori e problemi nella migrazione dalla versione 4.2 alla 4.3

    I principali problemi di una mancata migrazione sono:

    1. Impossibilità di leggere Posware.exe.config oppure reperire da esso la Connection string da usare per accedere al database cassa già presente.
    2. La versione 4.2 di Posware è corrotta e all'interno di "Installazione applicazioni" di Windows non è presente una voce per Posware.

    Nei prerequisiti di migrazione, tramite il tool PosUpdateCompanion, si verifica l'accesso al database cassa già esistente e la sua struttura per determinare se Posware può essere migrato alla v. 4.3, infatti il tool si aspetta che il database sia presente ed accessibile. Per verificare MySQL ed accedere al database cassa, i dati di accesso verranno presi dalla Connection string all'interno di Posware.exe.config (parametro Connessione negli applicationSettings). Se la Connection string non è corretta, il SetupPosware non procederà con l'installazione.

    Inoltre, il tool PosUpdateCompanion verifica anche che ci sia una versione di Posware non inferiore alla 4.2.306, la versione minima supportata per la migrazione alla v. 4.3 dell'applicativo. Se questa condizione non dovesse essere soddisfatta, il SetupPosware mostrerà un errore specifico e per risolvere il problema sarà necessario installare una versione di Posware 4.2 pari o superiore alla 4.2.306 prima di poter riprovare ad eseguire la migrazione.

    Aggiornamento dei file di configurazione

    A partire dalla versione 4.3, i file di configurazione di Posware, delle Connection strings e delle impostazioni utente sono stati riportati allo standard .NET rimuovendo l'approccio custom.

    SetupPosware provvederà ad installare (o migrare in caso di migrazione dalla v. 4.2) automaticamente i seguenti file di configurazione:

    • Posware.exe.config
    • ConnectionStrings.config
    • Posware.User.config

    I punti cruciali di questo processo sono:

    1. Le Connection strings sono state spostate nel file dedicato ConnectionStrings.config e non sono più presenti all'interno di Posware.exe.config
    2. Posware.exe.config se necessario può essere sovrascritto da PosUpdate durante un aggiornamento. Tutte le impostazioni di runtime .NET che richiedono un valore all'interno di Posware.exe.config, dovranno essere inserite nel nuovo file Posware.User.config che il runtime .NET caricherà automaticamente
    3. Tutte le impostazioni che prima erano sotto le voci Posware.MySettings (applicationSettings) sono ora state spostate nel database di posware in tabparametriextra.

    Alcune di queste impostazioni che sono state spostate in tabparametriextra sono:

    • campiRicArticolo
    • campiFiltroRicArticoli
    • messaggiServerGiacenza
