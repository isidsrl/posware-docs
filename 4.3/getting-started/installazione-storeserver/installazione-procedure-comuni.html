    Posware StoreServer

    Procedure comuni di installazione

    Prima revisione documento: 13 maggio 2024
    Ultima revisione documento: 28 agosto 2024

    Cenni preliminari

    Il presente documento, come introdotto nell'overview generale, descrive nel dettaglio la procedure comuni ad entrambi gli scenari di installazione dello StoreServer dando particolare enfasi ai vari passaggi necessari per portare a compimento queste operazioni.

    Backup preliminari

    Prima di iniziare tutta la procedura di installazione dello StoreServer, è assolutamente mandatorio eseguire i backup da usare in caso di malfunzionamenti e per l'eventuale procedura di rollback. Non eseguendo i backup, qualora si manifestassero problematiche di varia gravità che necessitino di rollback, quest'ultimo non sarà più possibile.

    Eseguire il backup di quanto segue:

    1. Backup dei database in uso quali cassamaster e posware
    2. Eventuali database e directory non direttamente collegati a Posware che possono essere influenzati dall'installazione di una o più procedure/dipendenze usate dalla nuova versione che si andrà ad installare (MySQL, SQL Server, runtime .NET, ecc...)

    Infine, se non fosse un'operazione troppo onerosa per il punto vendita e se ne avesse la possibilità e la disponibilità tramite procedure e hardware appositi, sarebbe ottimale anche un backup generale di sistema. Quest'ultimo backup non è obbligatorio, ma è solo una precauzione ulteriore che si può adoperare per contrastare eventuali effetti collaterali indesiderati.

    Installazione e configurazione del database

    MySQL

    Come introdotto nell'overview generale, in caso il provider del database scelto sia MySQL, è necessario installare MySQL v. 8.4, l'unica versione attualmente compatibile con lo StoreServer.

    Installazione

    Per prima cosa, scaricare l'MSI Installer per Windows a 64bit di MySQL v. 8.4 dal sito ufficiale e una volta completato il download procedere a lanciarlo.

    Andare avanti accettando i termini di utilizzo e proseguire fino allo step della scelta del tipo di setup, dove sarà necessario selezionare "Complete".

    A questo punto sarà possibile proseguire con l'installazione effettiva di MySQL v. 8.4. Una volta completata l'installazione, selezionare la checkbox "Run MySQL Configurator" prima di premere il pulsante "Finish", così che una volta chiuso l'installer partirà subito dopo il tool per configurare MySQL v. 8.4 nel dettaglio.

    Configurazione

    MySQL Configurator

    Una volta apertosi MySQL Configurator, si potrà procedere a configurare passo passo il provider del database:

    1. Il primo step sarà quello di scegliere la directory di MySQL v. 8.4, che va impostata pari a "C:\ProgramData\MySQL\MySQL Server 8.4\" in caso quella consigliata dovesse differire. In caso sul computer sia installata un'altra istanza di MySQL oltre alla versione 8.4 in questo passaggio sarà assolutamente necessario scegliere l'opzione Configure this server instance as a side-by-side installation, altrimenti verrà sovrascritta l'altra istanza rendendo così poi impossibile in caso di necessità eseguire la procedura di rollback.

    2. Nello step "Type and Networking" impostare il Config Type pari a Dedicated Computer e lasciare come tipo di connessione del database la checkbox "TCP/IP", assicurandosi però di cambiare la porta ad una non in uso da altre versioni di MySQL precedentemente installate. Infine selezionare la checkbox "Show Advanced and Logging Options".

      NOTA BENE: Se è necessario cambiare la porta in una non in uso da altre versioni di MySQL, negli step successivi dell'installazione assicurarsi di impostare adeguatamente la Connection String al database, in particolare nel file di configurazione dello StoreServer e in tutti i servizi legacy dove è impostata tale porta.

    3. Nella sezione "Accounts and Roles" del MySQL Configurator, impostare la password dell'account root a quella comunemente usata in Posware e procedere poi a creare un utente "isid" con ruolo DB Admin assegnandogli sempre la stessa password dell'account root.

    4. Nello step "Windows Service" lasciare tutte le impostazioni di default, ossia la checkbox "Configure MySQL Server as a Windows Service" selezionata, il Windows Service Name deve essere MySQL84, la checkbox "Start the MySQL Server at System Startup" anch'essa selezionata ed infine la voce "Run Windows Service as ..." deve essere pari a Standard System Account.

    5. Nella sezione "Server File Permissions" lasciare selezionata la voce "Yes, grant full access to the user running the Windows Service (if applicable) and the administrators group only. Other users and groups will not have access.".

    6. In "Logging Options" lasciare selezionate soltanto le due checkbox "Slow Query Log" e "Binary Log", mentre la checkbox "General Log" non va selezionata.

    7. Nello step "Advanced Options" lasciare il Server ID pari a 1 e la voce "Table Names Case:" impostata su Lower Case (default).

    8. Saltare la sezione "Sample Databases" non selezionando alcuna voce.
    9. A questo punto si potrà applicare la configurazione impostata a MySQL v. 8.4, concludendo così le operazioni del tool MySQL Configurator.
    Configurazione dei parametri del file my.ini e del metodo di autenticazione

    Terminata la configurazione del provider tramite il MySQL Configurator, sarà necessario impostare alcuni parametri all'interno del file my.ini di MySQL, situato nella directory di MySQL v. 8.4 ("C:\ProgramData\MySQL\MySQL Server 8.4\"). Prima di procedere però con la modifica del file assicurarsi di stoppare il servizio MySQL84 di Windows.

    I parametri da impostare sono i seguenti, tenendo presente come regola generale, che se il parametro è già presente nel file va semplicemente cambiato il valore, mentre se non è presente vanno direttamente aggiunti sia il parametro che il valore:

    • max_connections = 500
    • lower_case_table_names = 1
    • mysql_native_password = ON
    • authentication_policy = 'mysql_native_password,'
    • log_error_suppression_list = MY-013360
    • key_buffer_size = 64M
    • innodb_dedicated_server = ON

    Una volta completata la configurazione di questi parametri, procedere a riavviare il servizio MySQL84 di Windows. Inoltre, in caso sulla macchina dello StoreServer si utilizzi una vecchia versione di Navicat per interfacciarsi con il database, è necessario sostituire il nuovo metodo di autenticazione di MySQL, caching_sha2_password, con il metodo legacy, mysql_native_password, altrimenti non sarà possibile connettersi ai database in MySQL v. 8.4 con Navicat.

    Per fare ciò, connettersi tramite il MySQL Command Line Client alla propria connessione MySQL e lanciare i seguenti comandi, dove al posto di your_new_password va inserita tra virgolette la password comunemente usata in Posware:

    ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'your_new_password';
    FLUSH PRIVILEGES;

    Server non dedicati

    Lo StoreServer è progettato per funzionare su server dedicati. In caso contrario, bisognerà agire manualmente sulla memoria riservata a MySQL per evitare che utilizzi tutta la memoria esistente.

    Impostare, quindi, ad OFF la variabile innodb_dedicated_server nel file my.ini e fare riferimento alla guida di MySQL per impostare la quantità di RAM da dedicare con questi valori minimi:

    • 1 Gigabyte: fino a 2 casse oppure fino a un massimo di 1,5 milioni di incasso.
    • Almeno il 50% di RAM: in tutti gli altri casi.

    Importazione del database del cliente in MySQL v. 8.4

    Dopo aver completato l'installazione e la configurazione di MySQL v. 8.4, sarà necessario esportare il database posware del cliente dalla vecchia versione di MySQL ed importarlo nella 8.4, così che sarà pronto per essere migrato, in seguito, con il Database Upgrader durante l'installazione dello StoreServer. Durante l'importazione del database in MySQL v. 8.4 si raccomanda vivamente di rinominare il database da posware a posware42, così da non dover svolgere questa operazione successivamente.

    NOTA BENE: Si ricorda che da questa installazione in poi è obbligatorio usare la funzionalità apposita messa a disposizione da MySQL, ossia mysqldump, per eseguire l'esportazione e l'importazione dei dump dei database in MySQL. Non è più possibile, quindi, usare Navicat o altri strumenti per fare queste operazioni, altrimenti non verrà garantita la completa integrità del database durante gli spostamenti tra versioni diverse del provider. Inoltre, nell'esportazione del database con mysqldump è assolutamente mandatorio usare nel comando apposito "-r" invece del maggiore (>), così da effettuare il dump binario, che evita possibili errori che possono avvenire nel ripristino successivo del dump a causa di conflitti con la codifica.

    SQL Server

    Come introdotto nell'overview generale, in caso il provider del database scelto sia SQL Server, è possibile scegliere una tra le seguenti versioni, tutte compatibili con lo StoreServer: SQL Server 2016, 2017, 2019 o 2022.

    Installazione e configurazione

    Per la scelta della versione di riferimento, l'installazione e la configurazione fare riferimento alla documentazione di Microsoft riguardante SQL Server.

    Alcune configurazioni necessarie che andranno impostate sono le seguenti:

    • Scegliere come modalità di installazione Download Media col package Express Core, perché così si potrà effettuare un'installazione in cui sarà poi possibile configurare il provider step per step, rispetto alle installazioni di default. Questo serve per configurare le impostazioni successive.
    • Nello step in cui bisogna scegliere la configurazione dell'istanza di SQL Server scegliere Default instance.
    • Nello step della configurazione dell'engine del database impostare l'Authentication Mode pari a Mixed Mode (SQL Server authentication and Windows authentication) e settare la password dell'utente sa di sistema a quella comunemente usata in Posware.

    Importazione del database del cliente nella nuova versione di SQL Server

    Dopo aver completato l'installazione e la configurazione della versione scelta di SQL Server, sarà necessario esportare il database posware del cliente dalla vecchia versione di SQL Server ed importarlo nella nuova, così che sarà pronto per essere migrato, in seguito, con il Database Upgrader durante l'installazione dello StoreServer.
